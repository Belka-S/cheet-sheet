JavaScript

Today, using JavaScript you can create:

  • web applications using frameworks such as React, Vue, Angular and others;
  • back-end applications using Node.js;
  • mobile applications with React Native or Ionic;
  • desktop applications using Electron;
  • microcontrollers with Johnny-Five and Espruino.

Adding Script

To add a script to a web page, HTML files use the script tag. In its src attribute you should include a link to an external JavaScript file. The script should be added before the <body> closing tag, after all content.

<script src="./js/script.js" type="module"></script>

In order to run your script in strict mode, it is enough to enter the following directive at the beginning of the js-file. Another way to run script in strict mode to use attribute type="module". Always write your code in strict mode.

Syntax Basics

  • Statement is a related set of words and symbols from the syntax of a language that combines to express a common idea, in order to give an instruction to a PC.
  • Expression is a reference to a variable or value, or to a set of variables and values in combination with operators. The statement contains expressions.
  • Interface is a set of properties and methods of an entity available for use in the source code.
    • data has properties, for example, a string has a length. The syntax for accessing a property is very simple: use a dot - entity.property_name.
    • data has its own predefined methods, this is an action call. The syntax for calling a method is similar, but parentheses are added at the end - entity.method_name().

Variables

Variables are used to store data and consist of an identifier (name) and a memory space for storing their values. A variable declaration begins with the keyword let or const:

  • use const by default, as most variables should be declared this way.
  • use let if you need to assign a different value to your variable during script execution.

Undefined is a variable that has been declared with the keyword let, but has not been initialized with a value.
Undeclared or not defined is a variable that has not been declared in the available scope.

Variable scope is the availability of variables in a certain part of code. Any construct using curly braces {} (conditions, loops, functions, etc.) creates a new local scope, and variables declared in this scope using let or const are not available outside this block.

Primitive types

typeof operator is used to get the variable's value type. It returns in place of its call the value type of the variable specified after it. A variable is not associated with any data type; its value has a type.

  • number - integers and floating-point numbers.
  • string - indexed set of zero or more characters, enclosed in single double or back quotes;
  • boolean - logical data type, true/false flags;
  • null - special value that essentially means nothing, is used in situations where emptiness must be explicitly indicated;
  • undefined - when a variable is declared but not initialized, its value is undefined.

User Input

  • data output methods are console.log() and alert(), the alert() method displays a modal window, with a text corresponding to the value of the variable (or literal) in parentheses;
  • data get from the user methods are prompt() and confirm(), as a result of their execution, they return what was entered by the user, it can be written to a variable for further use (prompt always returns a string).

Basic Operators

  • Mathematical operators +, -, *, /, % are operators, whereas their objects are operands;
  • Comparison operators > and < (greater than/less than),>= and <= (greater/less than or equal), == and !=(equal/not equal), === and !== (strict equality/inequality);
  • Equality operators - the "non-strict" equality operators == and != convert types of compared values to numbers.

Numbers

  • Number() function - most operations and functions convert a value to a number automatically, use this function to do this explicitly;
  • Number.parseInt() method - parses an integer from the string;
  • Number.parseFloat() method - parses a fractional number from the string;
  • Number.isNaN() method - checks if the entered value is NaN or not;
  • toFixed() method - to add and round the result to a certain decimal place;
  • Math.floor() method - returns the smallest integer less than or equal to the specified one;
  • Math.ceil() method - returns the largest integer greater than or equal to the specified number;
  • Math.round() method - returns the value of a number rounded to the nearest integer;
  • Math.max() method - returns the largest number from a set;
  • Math.min() method - returns the smallest number from a set;
  • Math.pow() method - exponentiation;
  • Math.random() method - returns a pseudo-random number in the range (0, 1).

Strings

The content of a string cannot be changed, only read.

  • String() function - to convert a value to a string;
  • If you apply the + operator to a string and any other data type, the result of this addition is a string. This operation is called concatenation, or string addition. Template strings are an alternative to concatenation with more convenient syntax, they may contain placeholders with the dollar sign and curly braces - ${expression}
  • .length property - to get value of a string;
  • .toLowerCase() method - return a new string in the lower case without modifying the original string;
  • .toUpperCase() method - return a new string in the upper case without modifying the original string;
  • .indexOf('searchValue'[, fromIndex]) method - returns the position (index) of the first substring match, or -1 if nothing is found;
  • .includes('searchString'[, position]) method - checks if a substring is included in the string, returns true/false;
  • .endsWith(searchString[, length]) method - determines if a string ends with the characters (substring) specified in parentheses, returns true/false
  • .startsWith(searchString[, length]) method - determines if a string starts with the characters (substring) specified in parentheses, returns true/false
  • .replace(regexp|substr, newSubStr|function[, flags]) method - return a new string in which the first substring matches have been replaced with the specified value;
  • .replaceAll(regexp|substr, newSubstr|function) method - return a new string in which all substring matches have been replaced with the specified value;
  • .padStart(targetLength [, padString]) method - pads the current string from the start with another string until the resulting string reaches the given length;
  • .padEnd(targetLength [, padString]) method - pads the current string from the end with another string until the resulting string reaches the given length;
  • .at(index) method takes an integer value and returns a new String consisting of the single UTF-16 code unit located at the specified offset;
  • .substring(indexStart[, indexEnd]) method - returns the part of the string between the start and end indexes, or to the end of the string.
  • .trim() method - removes whitespace from both ends of a string and returns a new string, without modifying the original string.

Logical Operators

In logical operations, the types of operands are converted/cast to true or false. Truthy and Falsy are the terms used for those values that are converted to true or false in a logical operation, although they were not originally booleans. Values that are converted to false in logical conversion 0, NaN, null, undefined, empty string, false.

  • Boolean() function - to convert a value to logical true or false;
  • Logical AND && - result will be false, since at least one of the operands is converted to false;
  • Logical OR || - result will be true, since at least one of the operands is converted to true;
  • Logical NOT ! - replaces value with the opposite.
  • Nullish coalescing operator ?? - returns its right-hand side operand when its left-hand side operand is null or undefined, and otherwise returns its left-hand side operand.

Branches and Cycles (Lesson 2)

Branching

Branches are used to execute particular code depending on the condition. The principle is simple: the condition result is converted to a boolean, true or false, after which the program executes this or that branch.

  • if (condition) {//if body} statement - if the condition is converted to true, the code in curly braces of the if body is then executed;
  • if (condition) {//if body} else {//else body} statement - if syntax is extended, if the condition is converted to false, the code in curly braces after the else statement will be executed;
  • else if statement - enables you to add after else another if statement with a condition.

Ternary Operator

condition ? expression_if_true : expression_if_false - the ternary operator is used as a more syntactically concise replacement for the if ... else statement

Switch Statement

The scope of switch is limited to tasks with one common question (what to compare) and many answer options (what to compare with). For comparison, the strict equality operator === is used.

switch (value) {
case value: statements;
break;
case value: statements;
break;
default:
statements;
}

Loops

  • while (condition) {//statement} - pre-test loop is a loop that is executed as long as a condition specified before its beginning stays true;
  • do {//statement} while (condition) - post-test loop is a loop in which the condition is tested after executing the loop body;
  • for (let i = 0; i < max; i +=n) {//statements} - count-controlled loop in which a variable changes its value from a set initial value to an end value with a step, and for each value of this variable the loop body is executed once.
    • break operator - completely terminates the execution of the loop and transfers control to the string after its body;
    • continue operator - does not interrupt the entire loop, but only the current iteration.
      • const number = 10;
      for (let i = 0; i < number; i +=1) {
      if (i % 2 === 0) {
      break / continue;
      } }

Arrays (Lesson 3)

Array

Is a data structure for storing and manipulating collections of indexed values.

  • const arrayName = [element1, element2, element3] - declared with the opening and closing square brackets [], array elements are separated by a comma;
  • arrayName[index] - to access the values of array elements;
  • arrayName[index] = newElement - can be changed if you access them by index and assign a different value;
  • arrayName.length property - length of an array, that is, the number of its elements;
  • console.table(arrayName) - displays a table whith the indexes and values of the arrayName

Array Iteration

  • for (let i = 0; i < arrayName.length; i +=1) {console.log(arrayName[i]);} - to iterate over an array element by element;
  • for (const varName of arrayName/stringName) {console.log(varName);} - iterates over iterable (enumerable) objects such as arrays and strings;
    • for (const varName of arrayName/stringName) {if (varName === searchName) {console.log(varName); break;} } - after indentifying of searchName, the loop is interrupted, as there is no point in looking further
    • for (let i = 0; i < i arrayName.length; i +=1) {if (arrayName[i] < threshold); continue;} - for numbers less than the threshold value, continue is triggered and control is passed to the next iteration.

Value and Reference

  • Primitives are strings, numbers, booleans, null and undefined; when assigning, they are copied entirely, by value.
  • Complex types are variable to which an array or object is assigned does not store the value itself, but the address of its space in memory, they are copied by reference.

Array Methods

  • .split(delimiter) method - converts a string to an array by splitting it with delimiter, strArray = strArray.split(/\s+/) - unlimited space pattern;
  • .join(delimiter) method - joins the elements of an array into a string by joining it with delimiter;
  • .indexOf(value) method - returns the first index at which the element with value is found in the array;
  • .includes(value) method - checks if there is an element with value in the array, and it returns true or false;
  • .push(value1, value2, ...) method - adds one or more elements to the end of the array, without the need to specify the indices of the added elements;
  • .pop(value) method - removes the last element from the end of the array and returns the removed element, if the array is empty, it returns undefined;
  • .unshift(value1, value2, ...) method - adds one or more elements to the beginning of the array, without the need to specify the indices of the added elements;
  • .shift(value) method - removes one element from the beginning of the array and returns the removed element, if the array is empty, it returns undefined;
  • .slice(beginIndex, endIndex) method - returns a new array which is a partial copy from beginIndex to, but not including, endIndex;
  • .splice(position, num) method - removal, modifies the original array and returns an array containing the removed elements, where position indicates the position (index) of the first element to be removed and num indicates the number of elements to be removed;
  • .splice(position, 0, new_element_1, new_element_2, ...) method - addition, position argument specifies the starting position in the array where new elements will be inserted, second argument is 0, it instructs the method not to remove elements where new ones are added;
  • .splice(position, num, new_element_1, new_element_2, ...) method - replacement, position indicates the position (index) of the first element to be removed num determines the number of elements to be removed;
  • .concat(arrayName1, arrayName2, ...) method - consolidates two or more arrays into one;
  • .sort([(a, b) => a - b)]) method - sorts the elements of an array;
  • .reverse() method - reverses an array;
  • rest parameter syntax allows to apply any Math.methodName to an Array: Math.methodName(...arrayName).

Functions (Lesson 4)

Function declaration, expression, calling

Function is a subroutine, an independent piece of code designed to repeatedly execute a specific task with different initial values. Function name must be a verb answering the question "What to do?". If you need to declare a function with parameters other than undefined, it is enough to specify the default value.

  • Function declaration function functionName(parameters) {\\function body} - can be called before its creation;
  • Function expression const functionName = function(parameters) {\\function body} - cannot be called before its creation, only after, as it is literally a const variable declaration;
  • Function calling functionName(arguments) - in parentheses you can pass arguments, i.e. values for the declared function parameters.

Usage Features

  • arguments can be accessed using the special variable, which is only available inside the function and stores all arguments as a pseudo-array:
    function multiply() {let total = 1; for (const varName of arguments) total *= varName; return total;}
  • Array.from() method creates an array from a pseudo-array:
    function createArray() {const newArray = Array.from(arguments); return newArray}
  • rest parameter syntax allows a function to accept an indefinite number of arguments as an array:
    function fnName(...argToArray) {There is access to Array 'argToArray' that was made from arguments}
  • return operator passes a value from the function body to external code and exit the function:
    function multiply(x, y, z) {return x * y * z;};
  • return early pattern is a way to return early from a function using the return operator:
    function withdraw(amount, balance) {
    if(amount === 0) {
    console.log("Enter an amount greater than zero");
    return;}
    if (amount > balance) {
    console.log("Insufficient account balance");
    return;}
    console.log("Withdrawal has been successful");}

Scope

Scope is a mechanism that determines the availability of variables in your code. Scope chain: scopes form a hierarchy so that child scopes can access variables from parent scopes, but not vice versa.

  • Variables declared at the very top level, that is, outside of any constructs like if, while, for and functions, are in the global scope, and they are available everywhere after they are declared.
  • Variables declared inside if and for statements, functions and other blocks of code enclosed in curly braces {} are in the block scope, and they are available only within this block of code or those nested inside it.
  • Scope chain look-up - the interpreter first tries to find a variable in the scope in which such variable was accessed. If there is no such variable in the local scope, then the interpreter goes out, one level per attempt, until it finds the necessary value or reaches the topmost scope (global) and realizes that no variable with such an identifier can be found, since it does not exist

Call Stack

Call stack is a mechanism for keeping track of where the interpreter is located in the code that calls multiple functions. You can see which of the functions is currently being executed, which functions are called from within the function being executed, which will be called next, etc. Stack is a data structure using the LIFO (Last-In-First-Out) principle. The last thing that is added to the stack will be removed from it first, which means that you can add or remove elements only from the top of the stack.

Objects (Lesson 5)

Object properties and methods

Objects serve to describe and group the characteristics of an entity. Objects are also called dictionaries, that is, they contain terms (properties) and their definitions (values). Objects can store not only data, but also functions for working with this data, i.e. methods. If the property value is a function, such property is called an object method. Such objects can be called models. They couple data with methods to work with that data.

  • const object = {propertyKey: value, ...} - object creation, property value can be another object in order to store nested and grouped data;
  • const object = {variableName, ...} - shorthand object creation, when creating an object and you need to take the property value from a variable or function parameter of the same name as the property;
  • Object.create(proto[, propertiesObject]) method - creates a new object, using an existing object as the prototype of the newly created object;
  • Object.assign(target, source1, source2, ...) method - copies all enumerable own properties from one or more source objects to a target object;
  • object.propertyKey - to access a value of the proherty;
  • object["propertyKey"] - to access a value of the proherty;
  • object[variableName] - to access a value of the property with a name that you do not know, as it is stored as the value of a variable;
  • object.propertyKey.nestedPropertyKey - to access a value nested the property;
  • object.propertyKey = value - adding objects property whith value or changing objects property value;
  • delete object.propertyKey or delete object[propertyKey] - removes a property from an object;
  • const object = {propertyKey: value1, [variableName]: value2, ...} - to add a property with a name that you do not know, as it is stored as the value of a variable or as the result of a function, the value of a computed property can be any valid expression;
  • fnName: function() {\\Body} or fnName() {\\Body} - to store functions for working with this data;
  • this.propertyKey - to access the properties in object methods, we refer to it through this and then, as usual, to its properties using dot notation.

Iteration

  • for (const key in objectName) { \\ - to iterate over objects
    key; \\ - to access properties
    objectName[key];} \\ - to access values of the proherties;
  • hasOwnProperty(key) method - returns a boolean indicating whether the object has the specified property as its own property or inherited one;
  • Object.keys(object) method - returns an array of own property keys of an object;
  • Object.values(object) method - returns an array of its own property values
  • Object.entries(object) method returns an array, with each element as another array of the propertyName and the value of this property.

Array of Objects

To iterate over an array of objects of the same type, use the standard for...of loop. To get property values of each object, use dot notation, since the set of properties and their names are the same in each object, only their values differ.

Destructuring and spread/rest (Lesson 6)

Spread and Rest Syntax

  • ... (spread) passing arguments - to spread a collection of elements from array, string or object to the places where a set of single values is expected;
  • ... (spread) creating a new array - to create a copy of an array or "bond" an arbitrary number of arrays together to make a new one:
    const copyOfArray = [...array];
  • ... (spread) creating a new object - to spread the properties of an arbitrary number of objects to a new one:
    const third = {propertyName: value, ...first, ...second }, it is possible add properties to any location, spread order matters, object property names are unique, so the properties of the sprayed object can overwrite the value of an existing property if their names are the same;
  • ... (rest) collecting all function arguments - to collect a group of independent elements into a new array;
  • ... (rest) collecting part of function arguments - to collect into an array only the required part of the arguments:
    function fnName(firstNumber, secondNumber, ...otherArgs);

Object Destructuring

Destructuring enables you to "unpack" the values of an object's properties into local variables, if the variable and property names are the same, assignment takes place, otherwise it will be assigned undefined:
const {propertyName1, propertyName2, propertyName3} = objectName;

  • const {varName1, varName2, varName3 = value} = objectName - specifies default values,they will be assigned only if the object does not have a property with that name;
  • const {propertyName1: newPropertyName1, propertyName2: newPropertyName2} = objectName - changing variable name;
  • for (const book of books) {const {title, author, rating} = book;} - destructuring in loop body;
  • for (const {title, author, rating} of books) {} - destructuring in loop can be done right where the variable book is declared;
  • const {propertyName1, propertyName2: {nestedPropName1, nestedPropName2}, propertyName3} = objectName - deep destructuring is for the properties of nested objects.;

Array Destructuring

Destructuring assignment can be used for arrays as well: use square brackets [] instead of curly braces {}.

  • const [varName1, varName2, varName3] = arrayName - creates variables with elements placed in them in ascending order, from 0 to the array end;
  • let varName1, varName2, varName3; [varName1, varName2, varName3] = arrayName - value can be assigned to a variable after arrays declaration;
  • const [varName1, varName2, varName3 = value] = arrayName - specifies default values;
  • const [varName, ...restArrayName] = arrayName - unpacks and assigns the rest of the array elements to a variable;
  • const [, , varName] = arrayName - skips elements.

«Parameter Object» Pattern

If a function has more than two or three arguments, it is very easy to get confused about what is passed and in which order. The parameter object pattern solves this problem by replacing a set of parameters with just one, an object with named properties:

  • function fnName(objectName) {const {varName1, varName2} = objectName; \\Function body} - destructure the object in the objectName parameter;
  • function fnName({varName1, varName2}) {\\Function body} - destructure the object in the function signature.
