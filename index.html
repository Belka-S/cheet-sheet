HTML - structure and semantics, CSS - style and position, JS - functionality.

Websites are the backbone of the Internet, and they are all built using three main technologies.

  • HTML is used for filling web pages with content and for shaping their structure.
  • CSS is intended for styling, positioning and appearance.
  • JavaScript makes it possible to add functionality, that is, responses to user actions.

HyperText Markup Language HTML (Lesson 1-2)

Document Structure

Document consists of a tag "tree". Tags are elements of the Hypertext Markup Language, the smallest building blocks that make up any web page. There are paired (<tag_name>...</tag_name>) and single tags. They can be nested within each other.

Top-level tags are: <html>, <head>, <body>. All text placed inside the <title> tag is displayed in a browser. The <meta> tag is used to store information intended for the browser and search engines.

Attributes

Additional tag settings that you can use to change the properties and behavior of an element.

  • href - the address of the web page that will be accessed when clicking on the link text.
  • target - indicates in which tab the page will open when clicking on the link.
  • title - adds a tooltip to the link text.
  • src - URL.
  • alt - an alternative text that will be shown if the image is not loading.

Document flow

The vertical and horizontal order in which elements are displayed on a page. Block-level element occupies the entire line, regardless of its content length; therefore, several block elements visually follow each other from top to bottom.

Semantics

Approach to creating HTML markup based on the use of tags according to their outlined purpose in the specification. The markup of a web page should make sense, like a newspaper or magazine page. The page has sections, headings, lists, images, text paragraphs, etc. Semantic markup based on the type and structure of the content being marked up.

Text markup

  • <p> tag - generic container for grouping small phrase elements, separating them from each other and further styling.
  • <h1>...<h6> tag - group defines the text headings of semantic sections of various levels; they indicate the importance of the content section that follows them.
  • <ul> and <ol> tag - make it possible to organize collections and present them in an intuitive and user-friendly way. A list is a container, and only list items, the <li> tags, can be its children.
  • <a> tag - creates links or text, clicking on which you go to another page, download a file, etc. (href="https:/", href="tel:", href="mailto:" target="_blank", relrel="noreferrer noopener").To create an anchor link, you need to add the id attribute (a unique identifier) to the tag to which you want to scroll the page.
  • <button> tag - interactive element activated with JavaScript. lways explicitly specify the type attribute. It default value is submit, but most often you will need button.
  • <dl> (description list) tag - container for a list of pairs of related terms and their descriptions, that is, a dictionary. For marking up terms, use the <dt> tag, and <dd> for descriptions.
  • <span> tag - generic container for small text content, similar to <div>, this tag is non-semantic and is used for additional styling.
  • <pre> tag - enables you to add preformatted text with forced line breaks.
  • <br> tag - makes a line break
  • <up> and <sub> tags - superscript and subscriptcharacters to mark up simple mathematical formulas.
  • <time> tag - value of the datetime attribute in the ISO 8601 format.
  • <address> tag - stores contact information about the author of an article
  • <strong> and <b> tag - are used to highlight an important word or phrase within a text.
  • <em> and <i> tag - are used to emphasize parts of the text.

If a page contains text in various languages, you can make it more accessible for assistive technologies by assigning the lang attribute <html lang="ua"> not only to the entire document, but also to individual tags.

Images

<img> tag is intended for marking up images in various graphic formats.

  • src="path" - absolute path specifies the exact location of the file in the folder structure on the server, relative path describes the path to the resource relative to the current file.
  • alt="description" - a required attribute, an alternative description.
  • width="value" and height="value" - the image size in pixels.

Graphic formats. Raster or bitmap graphics describe a graphic file in the form of an array with each pixel coordinates and a description of each pixel color. Vector graphics (SVG, Scalable Vector Graphics) mean a set of rules or equations that define lines, along with additional properties, line color and background for shapes. Images make up around 90% of the total size of a website's resources, so they need to be optimized first.

Sectioning

Set of semantic tags for marking up large logical sections and improving the structural semantics of web pages:

  • <header> tag - highlights the introductory part of the entire page, a complex section or an article. It usually contains a logo, navigation, introductory text and complex heading.
  • <main> tag - main content that cannot be found on other site pages (unique). There can be only one such tag per page.
  • <footer> tag - final part of the entire page, its section or article. It often contains copyright, a list of social media links, contact information, etc.

Semantic sections:

  • <nav> tag - major navigation only, not for any group of links in the document. It is the section with links to sections of the current or other pages.
  • <article> tag - independent, standalone and meaningful in itself part of a document.
  • <section> tag - large section that combines related content.
  • <div> tag - generic, semantic-free container. It is used as a wrapper for further content styling.

Validation

Needed to find errors in the markup syntax and discrepancies between the document and the HTML specification. The validator checks the following items:

  • Correct tag spelling.
  • Relevance of tags and the possibility of their use in a given HTML version.
  • Correctness of tag closing and nesting.
  • Correct use of attributes.

Developer tools are built-in browser functionality that helps you get information about the page source code, styles, network requests and much more.

Tables

Make it possible to place content in cells. A table consists of rows with cells. Table sections:

  • <table> tag - the entire table and serves as a container for the content.
  • <thead> tag - header, wraps a group of table rows in the heading area.
  • <tbody> tag - body, wraps the main group of table rows.
  • <tfoot> tag - footer, defines a group of table rows in the footer area.
  • <tr> tag - table row. It cannot be empty and must contain at least one cell.
  • <th> tag - table header. It is a semantic tag for the header cell.
  • <td> tag - table cell (table data). It can only be used inside a row. Each table row must have the same number of cells.

To create a row with column headings, use the standard <tr> tag for a row and mark the cells up with a special tag, <th> (table header), a semantic tag for the header cell.

Semantically correct to use <caption>, a special table tag, which must be put inside the table as the very first element.

Some tables are designed to merge multiple cells horizontally or vertically. The rule of equal number of cells in rows and columns cannot be canceled, but can be circumvented. In order to expand a cell horizontally, use the colspan="" attribute, and the rowspan="" attribute to expand it vertically.

Character entities

Special strings that browsers convert to characters:

  • &entity; to inset a character using its entity name.
  • &#CODE10; to insert a character using its decimal code.
  • ODE16; to insert a character using its hexadecimal code.

Video and Audio

To mark up video content, use the <video>, tmark up audio content, use the <audio> paired tags.

  • src - specifies the URL of the video file.
  • controls - checked attribute, if present, video/audio controls are added
  • autoplay - checked attribute, if present, video/audio playback starts as soon as it is available
  • loop - checked attribute, if present, your video/audio will be repeated in a loop
  • poster - the URL of the image that is used as a poster if the video is not loading or is not playing yet.
  • width and height - set the height and width for video, the same as for images

The preload attribute specifies the audio/video preload mode. Its value specifies what information will be loaded.

Selectors and Color CSS (Lesson 3)

Syntax

A block of CSS code is called a rule, which consists of a selector and a group of declaration blocks in curly braces. Each declaration includes a CSS property name and a value, separated by a colon. Each declaration consists of two parts, which are called property: value;

Adding Styles

Three methods of implementing styling information to an HTML document:

  • Inline Style - right in the style attribute, they are impossible to scale, difficult to override and reuse, so they are used very rarely. (<p style="color:blue; font-size:24px;"></p>)
  • Embedded Style Sheet - CSS rules are added inside the <style> tag in the head of the document. (<style type="text/css"></style>)
  • External Style Sheet - easy to scale, maintain and reuse on other pages. (<link rel="stylesheet" href="./css/styles.css" />)

Selectors - used to select one or a group of elements.

  • Element Selector (tag) - Describes the type of element that the CSS rule will be applied to.
  • Class Selector - main selector in modern development, used with the global class attribute. An element can have multiple classes, and in this case they are written using a space.
  • ID selectors tags can be set to a global attribute id, value of id must be unique on the page.
  • Attribute Selector - apply styles to elements with a specific attribute or value element[attribute]element[attribute="value"] or element[attribute="value"]
  • Descendant Selector - applies styles to all elements that are descendants of a specified element at any depth of nesting .menu li{}.
  • Child Selector - allows to select all elements that are the direct children of a specified parent element .menu > li{}.

State pseudo-class - used to apply styles to interactive elements when a specific event occurs

  • :hover - is activated when the mouse cursor is within an element
  • :focus - is activated when an interactive element (link, button, form field) receives a focus when navigating the page with the keyboard (Tab key)
  • :active - occurs when an item is activated.
  • :visited - is activated on links that have already been visited.

Property cursor sets the type of the mouse pointer when it is within the element .bottom {cursor: pointer;}.

Working with color

  • colour - change the color of the element’s text.
  • background-color - background color of an element.

RGB - by mixing red, green and blue in different proportions, you can get any other color.

Hexadecimal - based on the hexadecimal notation of a number (0 - the smallest fraction of color, f - is the largest fraction of color).

Color transparency - main way to add transparency to a color is with the rgba() function.

Style cascade

Cascading is a mechanism that controls the final property values of an element when multiple CSS rules are applied to it. The specificity value consists of four ranks.

  1. Embedded styles add one to the first rank, have the highest specificity, and cannot be overridden through CSS.
  2. Each identifier id selector adds one to the second rank.
  3. Each class, attribute, and pseudo-class selector adds one to the third rank.
  4. Each selector of an element and a pseudo element adds one to the fourth rank.

The specificity of a rule can be enhanced by using the keyword !important, if you add it after the property value.

Inheritance is a mechanism by which the values of some properties are passed from an ancestor to its descendants. Not all elements inherit property values from their ancestors.

CSS Variables - containers for values that are defined in a CSS document for repeated use.

Variable declaration begins with a double dash -- followed by its name in arbitrary notation :root {--brand-color: #3f51b5;}, :root selector is more specific then the <html> element.

To get access to the value of a variable, the built-in var() function is used. Variables can be overridden locally with other values. To do this, you need to declare a variable with the same name.

Text and Type CSS (Lesson 4)

Text formatting

  • text-decoration: none | underline | line-through | overline;
  • text-transform: none | uppercase | lowercase | capitalize;
  • text-align: left | right | center | justify;
  • line-height: multiplier | value | percent | normal | inherit;
  • letter-spacing: value | normal | inherit;
  • word-spacing: value | normal | inherit;
  • text-shadow: X offset, Y offset, blur radius, color;
  • white-space: normal | nowrap | pre | pre-wrap;
  • text-indent: value | percent | inherit;

Font properties

  • font-size: 16px; (by default)
  • font-weight: 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900;
  • font-style: normal | italic | oblique | initial | inherit;
  • font-family: serif | sans-serif | monospace;

Text Pseudo-Elements

  • ::first-letter {} - controls properties of the font, color, background, along with element geometry of the first character in the text of the element to which it is added.
  • ::selection {} - controls the selection style (color and background-color) for the text of an element, overriding the default values from the browser's style sheet.

Non-Standard Fonts

Google Fonts stores a large number of free fonts. All you need to do is find the font you like, get a link to it and link the font in your HTML file as another style sheet. Google Fonts returned us a typical CSS file, containing the @font-face rule, a font declaration sets font-family, font-style, font-weight, font-display, src, unicode-range.

Block Model CSS (Lesson 5)

Box Model

In the box model, any element is a rectangular container, regardless of how it is displayed on the page. The element has content, and it also has optional paddings, margins and a border.

width and height properties make it possible to explicitly set the total width and height of an element, or just the size of its content; it all depends on the type of a box model. Most of the elements should be set only with width. width and height at the same time should only be used for decorative elements with a fixed size such as icons.

box-sizing property sets the type of the element's box model box-sizing: content-box | border-box | inherit.

border-box is the standard box model in present-day projects. In order not to set the box-sizing property for each element separately, the following CSS code is used at the very beginning of the style sheet. *, *::before, *::after {box-sizing: border-box;}

Style normalization

We want HTML elements to look the same regardless of the browser. Style normalization is an attempt to solve this problem by a technique for improving the cross-browser compatibility of a web page.

Element Geometry

Element geometry has 4 sides: top, right, bottom and left. In this order, clockwise from the top.

  • padding property controls the padding between the content and the border of an element.
  • margin property controls the margins outside the border of an element. Vertical margins of adjacent block-level elements are not added, but the larger of the two is selected. The vertical margin of a nested box falls outside its parent and pushes both away from the sibling.
  • border property controls the width, style and color of the border of an element border: width style color;.

Element Types

In the browser style sheet, each element has its type, the display property.

  • Block elements- vast majority of elements (<h1>, <p>, <section>, <<ul>, <li>, <div>, <header>).
    • display: block property set.
    • height is determined by the content, unless explicitly set in the height property.
    • always start on a new line, so several adjacent block-level elements are positioned vertically one below the other.
    • can have any properties of the element's geometry: width, height, paddings, borders and margins.
  • Inline elements - positioned on one line one after another.
    • display: inline property set.
    • width and height properties have no effect, depend only on the content and cannot be explicitly set.
    • stay on a line as long as there is enough space on it and when the line ends they jump to the next line and fill it in.
    • can only be set horizontal geometry (left and right paddings, margins and borders), ignore the top and bottom ones values.
  • Inline-block elements - hybrid between block-level and inline elements, which has the best of both.
    • display: inline-block property set.
    • width and height depend on the content, but you can explicitly set the width and height properties.
    • stay on a line as long as there is enough space on it, after which new ones are carried over into the next line.
    • have any properties of the element's geometry: width, height, paddings, borders and margins.

Inline and inline-block elements have a right or, in the case of images, a bottom gap. This is not a margin or padding, but a blank space. The size of this gap depends on the text size (1/4 of the nearest ancestor’s font size)

Structural Pseudo-Classes

A way to select specific elements from a collection of children (siblings), without assigning additional classes:

  • :first-child and :last-child
  • :not(selector)
  • :nth-child(an+b) - n-th (n), even and odd (even/odd), from n-th (n+b), up to n-th (-n+b) element
  • :nth-last-child(an+b)
  • :first-of-type, :last-of-type and :only-of-type
  • :nth-of-type(an+b) and :nth-last-of-type(an+b)
  • :only-child
  • :empty

Flexbox CSS (Lesson 6)

Flexible Box Layout

Defines a set of properties for positioning, aligning and distributing space between items in a container, even if their size is unknown or dynamic.

The basic idea is to separate elements into a box container (flex container) and its children (flex items). A Flexbox container can change the width and height of its children, as well as their placement direction: alignment in a column or row, the order in which items are displayed and the spacing between them.

The direction of elements is determined by container axes along which its items line up:

  • main axis is the flex container's main axis along which all items are laid out. It is not necessarily horizontal, as its direction is controlled by the flex-direction property. Container items are always positioned from main-start (the beginning of the main axis) to main-end (the end of the main axis).
  • cross axis is the container's cross axis running perpendicular to the main axis. Its direction depends on the main axis. Cross-start and cross-end are the beginning and end of the cross axis along which the rows of items are placed.

Container Properties

The properties of containers control the direction of the axes, multiple lines and item positioning on the line.

  • display: flex | inline-flex; - sets flex context for all children (not descendants) of the container, turning them into flex items.
  • flex-direction: row | row-reverse | column | column-reverse; - sets the direction of the main axis, determining the direction of items in the container.
  • justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly; - controls the positioning of items on the main axis, from main-start to main-end.
  • align-items: stretch | flex-start | flex-end | center | baseline; - controls the arrangement of items along the cross axis.
  • flex-wrap: nowrap | wrap | wrap-reverse; - enables you to turn a single-line container into a multi-line one.
  • align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch; - controls the alignment of all lines in a multi-line container along the cross axis if there is free space.

Item Properties

Flex items are children of flex containers, nesting level one items.

  • flex-basis: auto | value; - determines the initial item size before any available space is distributed.
  • flex-grow: proportion; - by default, all items are set to 0, that is, they do not try to take up additional free space even if available.
  • flex-shrink: proportion; - by default, all items are set to 1, that is, they will shrink as needed.
  • align-self: auto | flex-start | flex-end | center | baseline | stretch; - enables an item to change its position on the cross axis by overriding the value of the align-items property from the container.
  • order: position; - change the order of the items along the main axis.

Decorative Effects and Elements CSS (Lesson 7)

Image Types

  • Content images are semantically charged, they are part of the page content and contribute to telling the whole story. They are added in HTML markup using the <img> tag with a required alt attribute describing the image.
  • Decorative images are used solely for decoration. They convey no meaning and just enhace the appearance, can be added via CSS as a background using the background-image property, or marked up in HTML with the <svg> tag.

Element Background

  • background-color: color; - value can be a reserved word or a color in RGB, RGBA and HEX formats.
  • background-image: url('image URL'); - the address can be either a relative or an absolute path.
  • background-clip: border-box | padding-box | content-box | text; - by default, it repeats horizontally and vertically.
  • background-repeat: repeat | repeat-x | repeat-y | no-repeat; - by default, it repeats horizontally and vertically.
  • background-position: x y; - can be used absolute or relative units and keywords (top, bottom, right, left, center) as values.
  • background-size: auto | value | cover | contain; - enables you to adjust the size of your image by specifying the required width and height.
  • background-attachment: scroll| fixed | inherit; - by default, the background image scrolls with the element's content.
  • background: background-color background-image background-repeat background-position background-attachment; - if any component is not specified, its default value will be used.

An element can have multiple background images at the same time. It is enough to list them in the background-image property separated by commas. For each image, you can also set other background properties, also separated by commas in each property.

  • background-image: url('image-1'), url('image-2');
  • background-size: 100px, cover;
  • background-position: top right, center;

Content Image Position

In some cases, the content image is larger or smaller than the container it needs to be placed in, or the image has a different aspect ratio.

  • object-fit: fill | contain | cover | scale-down | none; - crops, stretches or scales your content image/video.
  • object-position: position | initial | inherit; - similar to background-position, but for content images. The default is 50% 50%, that is, vertically and horizontally centered.

Pseudo-Elements

Are used to add decorative effects (icons, spacers, overlays) without the need to create additional empty tags in the markup.

  • ::before creates a pseudo-element before all the element's content (at the beginning).
  • ::after creates a pseudo-element after all the element's content (at the end).

content property is required property. If you do not need any text content, its value must be set to an empty string, otherwise the browser will not create a pseudo-element.

:hover pseudo-class is used to change the styles of a pseudo-element when an event occurs on a parent element.

Gradient

A gradient in CSS is a background image, so it can be set in the background-image property. By default, gradient colors are evenly distributed, but this can be controlled using a color stop, a transition point where one color changes to another. The value can be set in pixels or percentages.

  • background-image: linear-gradient (to top left, red 15%, blue 40%, green 65%); - linear whith color stop, direction can also be specified using the angle in degrees, the deg unit.
  • background-image: linear-gradient (to right, #red 15%, blue 15%, blue 40%, green 40%); - solid gradient whith a sharp transition between colors.
  • background-image: radial-gradient (ellipse at center, #red 15%, blue 15%, blue 40%, green 40%); - radial gradient shape is ellipse.
  • background-image: radial-gradient (circle at top right, #red 15%, blue 15%, blue 40%, green 40%); - radial gradient shape is circle.
  • background-image: repeating-linear-gradient (to bottom right, red 20px, blue 20px, green 40px); - can be linear or radial.

Element Shadow

Add a three-dimensional effect and visual height to elements, can be outer and inner, blurred and flat, single and multiple. By default, the shadow size is the same as the element size, and its color is the same as the text color.

  • box-shadow: x-offset y-offset blur spread color; - using a negative spread radius , you can shrink the box shadow and offset on one side only.
  • box-shadow: inset x-offset y-offset blur spread color; - inner shadow.
  • box-shadow: 0 3px 2px red, 0 6px 5px blue, 0 12px 10px green; - multiple shadows

Vector graphics CSS (Lesson 8)

SVG (Scalable Vector Graphics)

Graphics format and language for describing vector images wich have a number of advantages.

  • Scaling - unlike raster graphics, SVG does not lose quality when scaled.
  • File size - vector images are much smaller than raster images when used for icons or other abstract graphics.
  • Responsiveness - using CSS and JavaScript, you can change the parameters of a vector image, such as color or borders.

SVG elements

Any vector image consists of a set of shapes that are represented by SVG elements.

<svg width="400" height="300" style="outline: 2px dashed #2a2a2a;"> <shape-markup> </svg>
  • <rect> - creates a rectangular area.
  • <circle> - reates a circle with a radius set using the r attribute.
  • <ellipse> - creates an ellipse with the rx and ry attributes.
  • <line> - creates a straight line whith x1, y1 attributes of the starting point, and x2, y2 of its end.
  • <polygon> - escribes a polygon with appearance determined by the points attribute
  • <path> - universal element for shapes of any complexity, SVG icons exported by this element, d attribute contains a set of coordinates and their connecting curves that set the direction of the lines.

SVG document

Vector graphics are stored in .svg files. These are text files with the <svg> tag, inside which the shapes that the browser will draw are described. Within one file, you can describe many groups of shapes using the <symbol> element. Give each symbol a unique id to later access any of them from the HTML document.

  • SVG-viewport - attribute of <svg width="400" height="200"> element, visible part of the infinite canvas for SVG document.
  • SVG-viewBox - attribute of <svg viewBox="x y width height"> element, window through which you can view some parts of SVG document.

Usage

There are several ways to use vector graphics, each with their own pros and cons.

  • Element <img> - it is enough to specify the path in src="" attribute to the image, there is no way to dynamically change the color or other characteristics of a vector image via CSS.
  • Element background - need to to specify property background-image: url(""), there is no way to dynamically change the color or other characteristics of a vector image via CSS.
  • Inline SVG - can be added to HTML as a whole by using <path d="> element, but inline SVG code increases the size of HTML files.
  • SVG sprite - need to add the markup of the <svg> tag and the nested <use> element with the href="" attribute, which specifies the path to the icon that has an anchor containing the symbol identifier in the sprite. <use href="./path-to-svg-sprite/sprite-name.svg#symbol-id"> </use>

Shape characteristics

In CSS can be changeed some characteristics of an inline vector image or icon from an SVG sprite, including fill, color or stroke type.

  • fill property - if no fill color is specified, the shape will be filled with the current text color.
  • fill-opacity property - controls only the opacity of the fill color.
  • stroke property - sets the stroke color, has no value by default.
  • stroke-widt property - controls the stroke weight. The default is 1 px.
  • stroke-opacity property - controls opacity of the stroke color to set the value from 0 to 1.

Positioned Items CSS (Lesson 9)

Positioned Elements

Static positioning is the initial state of all web page elements. Flexbox and the box model place elements in this static page flow. The relative, absolute, and fixed positioning methods allow you to take the element out of the page flow and manually position it using a coordinate system.

The position property is used to change the positioning type of the element. By default, the static value is set. The properties top, left, bottom, and right are used to specify the element’s position.

position: static | relative | absolute | fixed | sticky | inherit; An element with a position value other than static is called a positioned element.

  • relative - elements stay in the web page flow, and neighboring elements do not take their place, visually the element can be moved relative to its original position and can overlap its neighbors.
  • absolute - elements is taken out of the page flow, and the neighboring elements take its place, top, left, bottom, and right values for an element with absolute positioning are calculated relative to the nearest descendant not using static positioning.
  • fixed - elements are removed from the page flow and positioned relative to the viewport (browser window), they will be fixed to one position when scrolling.
  • sticky - while the element is in its container and the container does not leave the viewport’s scope, it will behave as if it was relatively positioned; as soon as part of the parent container goes out of the viewport’s scope, the element is fixed on the page as long as there is at least some part of the container on the screen.

Positioned elements are removed from the flow, which means that they need something that defines which element is located higher or lower on the z-axis, perpendicular to the plane of the screen. This is controlled by the z-index:{} property.

Overflow

Property overflow: visible | hidden | scroll | auto; - manages how the element’s content will behave if it exceeds the explicitly set length or width values of the element. Value auto is similar to the scroll value, but the scroll bars only appear where they are needed and are not shown immediately.

The property overflow with the value hidden is used when creating decorative effects that have hidden content appearing from outside the box’s borders (rounded images, decorative overlay).

Animation CSS (Lesson 10)

Transitions

CSS transitions allow you to animate the change in the value of a property over time, controlling the speed of that transition. The property's change occurs when a certain event happens, which is described by the appropriate pseudo-class. A transition always has only two states - the initial and final property value

  • transition-property: background-color | transform | etc.; - properties to which the transition effect will be applied, value can be a single property or a comma-separated list of properties.
  • transition-duration: Nms | Ns; - specifies the period of time during which the transition should take place.
  • transition-timing-function: linear | ease-in | ease-out | ease-in-out | cubic-bezier; - behavior of the animation, describes the speed of the property transition from one value to another.
  • transition-delay: Nms | Ns; - delay after which the transition animation will start.

All the ones can be combined into one composite transition property: body {transition: background-color 500ms linear, transform 500ms ease-in-out 500ms;}

Animations

Animations do not need a mandatory initiator (event); it can be repeated an infinite number of times, and there can be any number of intermediate states between the initial and final states. Of all the properties that can be animated, it is recommended to use only two, if possible: opacity and transform.It's all about how the browser renders the web page.

  1. JavaScript step - a script or some event, such as a hover or focus, that starts the animation of the element's styles.
  2. Style step - new element’s styles are calculated; specificity, conflicts, cascading and so on are calculated.
  3. Layout step - the geometry of the elements is calculated.
  4. Painting step - the layers are drawn.
  5. Composite step - compositing of all prepared layers and rendering the final image on the user's screen.

Animating the background property does not affect the geometry of the elements, so the browser just needs to follow the steps Paint > Composite, to animate opacity and transform properties, it only needs to perform the Composite step.

Animation declaration and properties

Animation is declared by the @keyframes changeColor {0% {color: red;} 50% {color: blue;} 100% {color: green;}} directive, which allows you to describe a set of animation frames (states), of which there must be at least two (initial and final).

To use an animation that has already been declared, you need to specify its properties.

  • animation-name: name; - to use an animation that has already been declared, you need to specify its name as a value for the animation-name property.
  • animation-duration: Nms | Ns; - sets the duration of the animation, the time in which all frames will be traversed.
  • animation-timing-function: linear | ease-in | ease-out | ease-in-out | cubic-bezier; - behavior of the animation.
  • animation-delay: Nms | Ns; - sets the delay before the start of the animation in seconds or milliseconds.
  • animation-iteration-count N | infinite; - sets the number of times the animation will be repeated.
  • animation-direction: normal | reverse | alternate | alternate-reverse; - controls the direction in which the animation plays, from start to end frame, from end to start frame, or alternating the order with each repetition.
  • animation-play-state: running | paused; - makes it possible to pause the animation.
  • animation-fill-mode: none | forwards | backwards | both; - determines what happens to the element's styles before the animation starts and after it ends.

Shorthand property that enables you to set animation more briefly - animation: name duration timing-function delay iteration-count direction fill-mode play-state;

2d-transforms

CSS transforms allow you to move, scale and rotate elements, that is, add decorative effects. Transformations change theshape and position of an element without affecting the rest of the page.

  • transform: none | scale (1.25, 1.40) | rotate (45deg) | translate (tx, ty) | skew (x-angle, y-angle) | perspective (400px); - esponsible for applying one or more transformations to an element.
  • transform-origin: top left; - property allows you to change the point of origin relative to which the transformation is applied. The default value is center, or 50%. This property can only be set for transformed elements.

By default, objects are rendered in 2D space. With transformations along the X and Y axes, for example rotate, an object looks distorted and unnatural. By setting perspective function you instruct the browser to render objects in 3D space.

Elements of Forms (Lesson 11)

Forms

Forms allow us to get information from the visitors of our website. Forms are "money pages" through which online stores sell their products, and online services get paid for the solutions they provide. A functional HTML form consists of two parts: The user interface (form elements) and the server (form processing).

Form elements:

  • <form name="signup_form" autocomplete="on" novalidate> - container for a group of related form elements.
  • <label for="user-email">Email</label> - links the description (label) to the interactive form element.
  • <input type="text" name=""/> - used to create input fields, type of the field is defined by the type="" attribute:
    • text - default value, single-line text field that accepts any character.
    • email - display a particular keyboard layout to the user with easy access to the @ symbol.
    • password - inputted password value will be visually hidden to protect the user's password, attributes minLength and maxLength allow us to limit the number of characters that the user can enter into the field.
    • radio - every radio button must have the same name="" attribute value, otherwise the will not know that they are part of a group, attribute checked specifies which button will be chosen by default.
    • checkbox - similar to radio buttons (must have the same name="" and can have checked attribute), however, they allow you to select any number of values.
    • number - used to allow only numbers, min and max attributes define the minimum and maximum values that can be entered.
    • tel - displays only numbers for inputting telephone numbers on mobile browsers.
    • range - used to create sliders with integer and fractional values,min and max attributes define the range of permitted values, step controls the accepted interval.
    • date or time - allows to use browser built-in calendar, min and max attributes define minimum and maximum dates.
    If an input field has the autofocus attribute set, it automatically gets focus when the page is opened. With the use of the placeholder="" attribute, we can display text hints informing the user about the data that they must input in the field when the <input> is empty.
  • <textarea name="" rows="" cols="" placeholder=""<textarea> - creates a multi-line text field for inputting a large amount of text, can be stretched horizontally and vertically, to control the user's ability to change the size, the CSS property resize: both | horizontal | vertical | none; is used.
  • <select id="size" name="size"> <option value="s">Small</option> <option value="m" selected>Medium</option> </select> - drop-down with the name attribute, which contains a selection of <option> elements with the value attribute. Sometimes you need to break the list down into seperate, nonrelated groups. To achieve this, the <optgroup> tag is used.
  • <datalist id="color"> <option>Red<option> <option>Green</option> <option>Blue</option> </datalist> - create the list of predefined values that appears when typing in the associated text field, <option value="""> elements can also have the value attribute.

Field grouping

The <fieldset> element is a container for the grouping of several related form elements, and the nested <legend> fills the role of a group title. Related radio buttons and checkboxes should always be grouped. Other field types are grouped as needed. These elements have great semantics but limited formatting options. In most cases, the <div> tag with the accessibility attributes role and aria-labelledby is used for grouping form elements.

Form Validation CSS (Lesson 12)

Validation

If the wrong format is used in a form field, feedback is received in the form of an error. This is called form validation – checking of the validity of input data. There are two types of validation – client-side (before the form data is sent to the server) and server-side validation (after the data is sent to the server but before the data is saved in the database).

Validation attributes

  • required - marks the form field as mandatory for completion.
  • minlength and maxlength - limits on the number of symbols that can be used in a certain field, for example, for a password or a name.
  • min and max - can only be used in fields with the type="number | range | date"
  • pattern - allows you to specify a regular expression, according to which the field values will be checked.

State pseudo-class

A selection of pseudo-classes created specifically for form elements that don't affect other elements. They are used to check form fields according to the entered data's validity or whether it is mandatory to fill in.

  • :enabled and :disabled - apply to the elements <input>, <select> and <button> and can be used to prevent a form from being sent before all of the fields are filled in.
  • :checked - applies to radio boxes and checkboxes and allows you to select only the marked fields, when you select a checkbox, the label text turns blue.
  • :in-range and :out-of-range - usable with the elements <range>, <number>, and <date> if they have the min and max attributes specified.
  • :required and :optional - depends on the presence or absence of the attribute required, allows us to check the field and show the user that it must have valid data before it is submitted.
  • :valid and :invalid - allow you to select elements with valid or invalid input values, constraints are set with the attributes type="", minlength, maxlength, and pattern
  • :placeholder-shown - applied depending on the visibility of the placeholder value, after inputting text in the form field, the color of the border field can change to blue as soon as the placeholder disappears.
  • :focus-within - applied to the element when it or the elements within it gain focus, unlike :focus, which itself selects the element that gets focus, :focus-within allows you to select ancestors.

Pseudo-class composition .form-input:not(:placeholder-shown):required:valid {/* ... */} - by combining multiple pseudo-classes in one selector we can configure the selector to only apply when data is inputted into the field.

BEM Methodology CSS (Lesson 13)

CSS Methodologies

Effective organization of CSS code is the key to efficiency. The use of a methodology and the best practices increase development speed and simplify future support and refactoring of the code.There are a lot of different methodologies, for example, SMACSS, SUITCSS, Atomic CSS, BEM and others.

The BEM methodology (Block, Element, Modifier)

Was created by Yandex for the quick development of sites that need to be supported for many years. This methodology allows you to create an expandable and reusable interface blocks/components. The BEM methodology prohibits selector tags, IDs, or combined sectors of the type div.class1.class2. Block styles, elements, and modifiers are described as simple class selectors.

  • Block - functionally independent component that can be reused, can be placed in any part of the page or project and will still retain its use.
    • Can consist only of Latin characters, numbers, and dashes.
    • Should answer the question "What is it?"
  • Element - component of a block that has no standalone meaning and cannot be used separately from a block.
    • Must be formed using the block__element scheme.
    • Should answer the question "What is it?"

    The structure of the block's BEM classes and elements is always flat.

  • Modifier - additional class modifier that defines changes in a block or element's appearance, state, or behavior.
    • Should be simple, describing the change being made.
    • Must be formed in the format block--modifier or block__element--modifier.
  • Mix - combine several block and element classes under a single tag, they provides the ability to make one block an element of another block, it's class name consists of the names of the parent and child block using the format parent__child.

SASS preprocessor CSS (Lesson 14)

CSS preprocessor — an add-on that extends the CSS syntax to ease the writing, support and refactoring of code. Preprocessors allow the use of constructions that speed up and simplify the writing of code. The most popular preproccesors are SASS, LESS, and Stylus. The most popular preproccessor is SASS, and its syntax SCSS. Advantages: compatibility, rich functionality, project structure, clean code.

Compilation

By default, the SASS compiler creates a CSS file for every SASS file that it finds in the project. So that we only have one style file after compilation, (partials) are used — Normal SASS files, whose names start with an underscore, for example, _logo.scss. The underscore tells the compiler that this is a fragment and should not be compiled into a separate CSS file and that it will be used in another SASS file. After compilation, the main.css file will be created for main.scss, which we will connect to index.html.

To connect a fragment to another SASS file, @import is used.

Nesting rules

Similar to nesting tags in HTML, you can nest CSS selectors in SASS. Nesting allows you to place one rule inside another. The & symbol (ampersand) allows us to specify where the parent selector must be placed.

Functionality

  • variables - are only useable within the selector nesting level in which they are defined, declaring a variable is the $ sign before the name and its value after the colon.
  • addition, subtraction, multiplication, division - unlike the in calc() function, you cannot mix unit types in preprocessors.
  • @extend - is used inheriting (extending) already existing styles.
    • @extend .class-name; - adds the neseccary selectors to the list of rules with the inherited styles.
    • @extend %placeholder-name; - to extend the selection of styles that doesn't need a base selector.
  • @mixin - allows you to create pre-made sets of properties with different values.
    • @mixin name {/declaration/;} - with fixed values.
    • @mixin name ($variable) {/declaration/;} - with variable values.
    .header {@include bordered(green);} - mixin styles are added to a selector using @include, after which mixin will be called.
  • @each - loop that evaluates the code for each variable in the list.
    @each $animal in dog, cat, bird {.#{$animal}-icon {background-image: url('');}}, where $animal - local variable to be sequentially assigned values from a list: dog, cat, bird - list of animals.
  • maps (dictonaries) - allows us to store a set of related information in the format $colors: ('primary': #8e3329, 'accent': #d98328). The built-in SASS function map-get() is used to get the property value body {background-color: map-get($colors, 'primary');}.
  • vendor prefixes - special prefixes used by browser developers (vendors) for experimental CSS properties and values that are either not yet accepted into the standard, or not yet fully implemented into the browser, the SASS compiler with additional settings writes them automatically.

Adaptive Layout CSS (Lesson 15)

Media Queries

@media media-type and (media-feature) { /* A set of CSS rules to be applied to the document if the media type and expression conditions are met */}

A media query is a special CSS rule declared by using @media, which can be followed by the device media-type and (media-feature) expressions that check the characteristics of the device (for example, the width of the viewport). The rules found in media queries participate in the standard cascading when the element's final styles are composed.

  • media-type - is used to describe the kind of devices on which the webpage can be displayed.
    • all - means any device, is used by default.
    • print - for printers and devices that reproduce the document to be printed, a browser displaying the document in "preview" mode.
    • screen - for a device with a physical screen: smartphones, tablets, monitors, TVs, etc.
  • media-feature - allows to define the browser's viewport width.
    • min-width - pecifies the minimum width of the viewport.
    • max-width - specifies the maximum width of the viewport.

Logical operators

@media only|not media-type only|and|not (media-feature) {/*A set of CSS rules*/}
  • and - to associate multiple media functions with value checks.
  • , - to specify a set of expressions that will cause the execution of a media query if any of them are fulfilled (will be replaced by the or operator).
  • not - to cancel a media query, is added to the beginning of the media query and is applied to the whole query.

Viewport

The Viewport is the visible, rectangle part of the web page; not including the address bar, bookmark panel, and other browser utilities. Mobile browsers display the web page in a viewport wider than the physical device's screen size. Web pages tailored for viewing on different devices should contain the meta tag viewport in the <head> section. This tells the browser how it needs to control the size and scale of the Viewport.

<meta name="viewport" content="width=device-width, initial-scale=1.0" />
  • width=device-width - sets the width of the viewport equal to the physical size of the devices's screen (device-width).
  • initial-scale=1.0 - sets 1:1 ratio between CSS pixels and hardware pixels.

Layout types

In modern web design, new approaches are used when creating websites. They combine elements of rubber formatting and new, modern techniques like adaptive and responsive.

  • Responsive - has several display variations that can be smoothly transitioned, the elements and the container can be stretched like rubber.
  • Adaptive - has several display variations, the design is changed by hard-set jumps between breakpoints and cannot be changed between those jumps.

The Mobile-First Strategy

The Mobile-First strategy means that first, the mobile version is created, and then the tablet and desktop versions. There are significant advantages to this approach when designing a website:

  • One website - only one project for all devices, which reduces the amount of developers required.
  • Friendly interface - users see the most important page content first.
  • Loading speed - site loads faster on mobile devices because the amount of downloadable and displayable resources is reduced.
  • Search rating - Google doesn't only give priority to sites that are optimized for mobile devices, but also takes into account the time it takes to open the page.

Styles for mobile devices are basic styles outside of media queries.
@media screen and (min-width: 768px) {.element {/* Tablet styles */}} - media queries are added after breakpoint 768px, in which the essential basic styles are redefined. CSS rules inside media queries add new styles or redefine previous ones.

Adaptive Graphics CSS (Lesson 16)

Pixel density

Retina display is a marketing term used by Apple, which describes device screens with a high pixel density. On a retina display, the screen can fit more physical pixels since the individual pixel size is smaller.

Retina screens require specially prepared resources. Graphic retinization is a technique for preparing bitmap graphics (text and vector graphics (SVG) are displayed equally well on any screen).

  • Physical pixels - the smallest elements of any screen, each of which has its own color and brightness.
  • CSS pixels (device-independent pixels) - an abstract value independent of the device screen and used by the browser to calculate the size of the content on the page.
  • Bitmap pixels - the small parts that make up a bitmap image (png, jpg, gif, etc.), each pixel contains information about its color and location in the image coordinate system.

Responsive images

Responsive image is a term used for describing a selection of techniques used in HTML and CSS, helping the content and background images look just as good on any device, regardless of the screen size and pixel density.

  • Background images - to retinize a background image, you must set the size of the background image equal to the size of the element itself. There are media functions that allow us to define the screen's pixel density in CSSand redefine the approach to the background image.

    /* Redefine the path to the 2x image if the screen density is at least 2 */
    @media (min-device-pixel-ratio: 2), (min-resolution: 192dpi), (min-resolution: 2dppx) {.box {background-image: url('photo@2x.png');}}

  • Content images - to download images provided by the developers that suit the current device is used element:
    • <img> - allows us to download different versions of the same image.
      <img srcset="photo.jpg 1x, photo@2x.jpg 2x" src="photo.jpg" alt="description"/>
      • srcset attribute - defines a list of versions of the same image with different sizes.
      • x descriptor - tells the browser which image version is suitable for a particular screen density.
    • <picture> - is used if we need to download an image with a different format, a different geometry. Available images are listed in the <source> tags.

      • Allows browsers to download photo.webp, and at the same time provides the photo.jpg alternative for browsers that don't yet support webp.
        <picture>
        <source srcset="photo.webp 1x, photo@2x.webp 2x" type="image/webp" />
        <source srcset="photo.jpg 1x, photo@2x.jpg 2x" type="image/jpeg" />
        <img src="photo.jpg" alt="description" />
        </picture>

      • Allows browsers to download desk.jpg for desktops, and tab.jpg for tablets.
        <picture>
        <source srcset="desk.jpg 1x, desk@2x.jpg 2x" media="(min-width: 1200px) />
        <source srcset="tab.jpg 1x, tab@2x.jpg 2x" media="(min-width: 768px) />
        <img src="photo.jpg" alt="description" />
        </picture>
